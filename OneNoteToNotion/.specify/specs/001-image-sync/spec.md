# 功能规格：图片同步支持

## 问题陈述

当前 OneNoteToNotion 同步工具无法处理 OneNote 页面中的图片。当 OneNote 页面包含图片时，图片会被忽略，导致同步到 Notion 的内容不完整。

### 影响范围
- 包含截图的文档页面
- 包含流程图、架构图的技术文档
- 包含表格截图的数据文档
- 任何带有图片说明的笔记内容

## 用户故事

### 故事 1: 解析 OneNote 图片

**作为** 同步工具
**我想要** 从 OneNote XML 中提取图片数据
**以便** 能够将图片同步到 Notion

**验收标准:**
- [ ] 能正确识别 `<one:Image>` 元素
- [ ] 能提取图片格式（png, jpg, gif 等）
- [ ] 能提取 base64 编码的图片数据
- [ ] 能提取图片尺寸信息
- [ ] 能提取图片标题/说明文字

### 故事 2: 自动缩图处理

**作为** 同步工具
**我想要** 自动压缩超过限制的图片
**以便** 确保大图片也能成功同步

**验收标准:**
- [ ] 检测 base64 数据大小，不超过 5MB
- [ ] 超过限制时自动缩图至符合大小要求
- [ ] 缩图时保持图片宽高比
- [ ] 缩图后使用 JPEG 格式以获得更好压缩率
- [ ] 缩图失败时降级为占位符提示

### 故事 3: 映射到 Notion 图片块

**作为** 同步工具
**我想要** 将图片转换为 Notion API 格式
**以便** Notion 能正确显示图片

**验收标准:**
- [ ] 能生成 `data:image/{format};base64,{data}` 格式的 Data URI
- [ ] 能构造 Notion image block 类型的请求体
- [ ] 图片能正确显示在 Notion 页面中
- [ ] 图片位置与原文档保持相对顺序

### 故事 4: 错误处理与降级

**作为** 同步工具
**我想要** 优雅处理图片处理失败的情况
**以便** 同步不会因单张图片失败而中断

**验收标准:**
- [ ] 图片数据损坏时记录警告日志
- [ ] 缩图失败时插入降级提示文本
- [ ] 不影响其他内容的同步
- [ ] 诊断日志中记录失败原因

## 非功能性需求

### 性能
- 单张图片处理时间 < 2 秒
- 内存峰值增加 < 100MB
- 不影响整体同步速度

### 兼容性
- 支持的图片格式：PNG, JPEG, GIF, BMP
- 支持 OneNote 2013+ 格式
- 兼容 Notion API 当前版本

### 可维护性
- 代码遵循现有架构模式
- 添加单元测试覆盖图片处理逻辑
- 诊断输出包含图片处理信息

## 成功指标

- [ ] 包含图片的 OneNote 页面同步后 Notion 能显示图片
- [ ] 超大图片（>5MB）能自动缩图后成功同步
- [ ] 同步日志明确显示图片处理结果
- [ ] 无因图片处理导致的同步中断

## 澄清事项 (Clarifications)

### C1: 图片压缩策略

**问题**: 超过 5MB 限制的图片如何处理？

**决策**:
- 所有图片统一转换为 **JPEG 格式**（获得更好压缩率）
- JPEG 质量设置为 **80%**
- 如果压缩后仍超过 5MB，则进一步缩小尺寸
- 保持原始宽高比

### C2: 失败降级处理

**问题**: 图片处理失败时如何处理？

**决策**:
- 插入占位符文本：`[图片处理失败: {原因}]`
- 记录警告到同步日志
- 不中断整体同步流程

### C3: 图片位置

**问题**: 图片在 Notion 中的位置？

**决策**:
- 保持原文档中的相对位置
- 图片作为独立 block 插入到对应位置

### C4: 诊断日志

**问题**: 需要记录哪些图片处理信息？

**决策**:
- 处理成功时：记录原始格式、原始大小、处理后大小
- 处理失败时：记录失败原因、原始大小
- 输出到诊断目录的 semantic.txt 文件
