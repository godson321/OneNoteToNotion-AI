# 功能规格：附件同步支持

## 问题陈述

当前 OneNoteToNotion 同步工具无法处理 OneNote 页面中的文件附件。当 OneNote 页面包含 PDF、Word、Excel 等附件时，附件会被忽略，导致同步到 Notion 的内容不完整。

### 影响范围
- 包含 PDF 文档的笔记页面
- 包含 Office 文档的工作资料
- 包含压缩包的项目文件
- 任何带有文件附件的笔记内容

## 用户故事

### 故事 1: 解析 OneNote 附件

**作为** 同步工具
**我想要** 从 OneNote XML 中提取附件数据
**以便** 能够将附件同步到 Notion

**验收标准:**
- [ ] 能正确识别 `<one:InsertedFile>` 元素
- [ ] 能提取文件路径（pathCache 属性）
- [ ] 能提取文件名（preferredName 属性）
- [ ] 能提取文件大小（size 属性）
- [ ] 能通过 COM 接口读取文件二进制数据

### 故事 2: 文件上传到 Notion

**作为** 同步工具
**我想要** 将附件上传到 Notion 的 S3 存储
**以便** Notion 页面能直接下载/查看附件

**验收标准:**
- [ ] 能调用 Notion API 获取上传 URL
- [ ] 能上传文件到 AWS S3 预签名 URL
- [ ] 能处理大文件分块上传（如果需要）
- [ ] 上传失败时降级为占位符提示

### 故事 3: 映射到 Notion 文件块

**作为** 同步工具
**我想要** 将附件转换为 Notion API 格式
**以便** Notion 能正确显示文件链接

**验收标准:**
- [ ] 能构造 Notion file block 类型的请求体
- [ ] 文件链接能在 Notion 页面中正常点击
- [ ] 文件名正确显示在 Notion 中
- [ ] 文件位置与原文档保持相对顺序

### 故事 4: 错误处理与降级

**作为** 同步工具
**我想要** 优雅处理附件处理失败的情况
**以便** 同步不会因单个附件失败而中断

**验收标准:**
- [ ] 文件不存在或无法访问时记录警告日志
- [ ] 上传失败时插入降级提示文本
- [ ] 文件超过大小限制时给出明确提示
- [ ] 不影响其他内容的同步
- [ ] 诊断日志中记录失败原因

## 非功能性需求

### 性能
- 单个小文件（< 10MB）上传时间 < 5 秒
- 支持文件大小上限：100MB（Notion 免费版限制）
- 不影响整体同步速度（异步上传）

### 兼容性
- 支持所有文件类型
- 支持 OneNote 2013+ 格式
- 兼容 Notion API 当前版本

### 可维护性
- 代码遵循现有架构模式（解析 → 映射 → 同步）
- 添加单元测试覆盖附件处理逻辑
- 诊断输出包含附件处理信息

## 成功指标

- [ ] 包含 PDF 附件的 OneNote 页面同步后 Notion 能显示可下载的文件
- [ ] 大附件（> 50MB）能正确提示大小限制
- [ ] 同步日志明确显示附件处理结果
- [ ] 无因附件处理导致的同步中断

## 澄清事项 (Clarifications)

### C1: 附件获取方式

**问题**: 如何从 OneNote 获取附件数据？

**决策**:
- 使用 OneNote COM API 的 `GetBinaryPageContent` 或文件路径直接读取
- 优先尝试 pathCache 指向的本地文件路径
- 如果本地文件不存在，通过 COM 接口获取二进制数据
- 将文件数据暂存到内存或临时文件

### C2: Notion 文件上传流程

**问题**: 如何将文件上传到 Notion？

**决策**:
- 使用 Notion API "Upload files to S3" 流程：
  1. 调用 `POST /v1/files` 获取上传 URL 和文件名
  2. PUT 文件到 AWS S3 预签名 URL
  3. 使用返回的文件名创建 file block
- 参考官方文档：https://developers.notion.com/reference/upload-files-to-s3

### C3: 文件大小限制

**问题**: 超过 Notion 限制的文件如何处理？

**决策**:
- Notion 免费版：5MB 图片，其他文件 100MB
- 超过限制的文件降级为文本：`[附件超过大小限制: {文件名} ({大小})]`
- 记录警告日志，提示用户手动上传

### C4: 失败降级处理

**问题**: 附件处理失败时如何处理？

**决策**:
- 文件不存在：`[附件文件不存在: {文件名}]`
- 上传失败：`[附件上传失败: {文件名} - {原因}]`
- 记录警告到同步日志
- 不中断整体同步流程

### C5: 诊断日志

**问题**: 需要记录哪些附件处理信息？

**决策**:
- 处理成功时：记录文件名、文件大小、上传耗时
- 处理失败时：记录失败原因、文件路径
- 输出到诊断目录的 semantic.txt 文件
